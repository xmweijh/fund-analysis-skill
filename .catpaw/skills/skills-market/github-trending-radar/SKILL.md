# GitHub 流行雷达

## 数据获取

运行 `scripts/fetch_trending.py` 抓取 GitHub Trending 数据（纯标准库，无需依赖）：

```bash
# 基本用法 — 今日全语言 Top 10
python3 scripts/fetch_trending.py

# 按语言筛选
python3 scripts/fetch_trending.py --lang python
python3 scripts/fetch_trending.py --lang rust --since weekly

# 获取完整信息（含 README + 仓库详情）
python3 scripts/fetch_trending.py --limit 5 --readme --detail

# Markdown 格式输出
python3 scripts/fetch_trending.py --limit 10 --format markdown
```

参数说明：
- `--lang`: 语言筛选 (python/javascript/go/rust/typescript 等)，不传则全部
- `--since`: daily (默认) / weekly / monthly
- `--limit`: 返回项目数，默认 10
- `--readme`: 同时抓取每个仓库的 README（前3000字）
- `--detail`: 通过 GitHub API 获取详情（topics/license/issues 等）
- `--format`: json (默认) / markdown

## 分析流程

### Step 1: 获取数据

根据用户需求选择参数运行脚本。建议默认获取 `--limit 10 --readme --detail` 以获取充分信息。

### Step 2: 筛选机会

从抓取的项目中，用以下标准筛选**适合独立开发者做成产品的机会**：

**入选标准** ✅
- 解决明确的用户痛点（不是纯学术/实验）
- 1-2 人可在 2-4 周内做出 MVP
- 有清晰的目标用户群（能描述出是谁、在什么场景下用）
- 存在付费意愿的场景（B端工具、效率提升、专业需求）

**排除标准** ❌
- 大厂核心基础设施（需大量资源维护）
- 纯学术论文复现（没有产品形态）
- 已有大量成熟竞品且无差异化空间
- 纯娱乐/彩蛋类项目

### Step 3: 生成机会卡片

每个入选项目生成一张机会卡片，包含 7 个维度：

1. **项目概要** — 名称、语言、Stars、一句话描述
2. **目标用户** — 谁会用？在什么场景下？痛点是什么？
3. **核心功能** — 项目解决了什么问题？关键能力是什么？
4. **变现思路** — 怎么赚钱？(SaaS订阅/一次性付费/Freemium/API计费)
5. **差异化方向** — 和现有竞品相比，可以从哪个角度切入？
6. **MVP 建议** — 最小可行产品应该包含什么？预估工期
7. **风险提示** — 技术门槛、市场风险、竞争风险、许可证限制

### Step 4: 综合评分

对每个机会从以下维度打分 (1-5)：

| 维度 | 说明 |
|------|------|
| 市场需求 | 目标用户群大小、痛点强度 |
| 实现难度 | 技术门槛、开发周期 |
| 变现可行 | 付费意愿、定价空间 |
| 竞争优势 | 差异化空间、护城河 |
| 趋势热度 | Star 增速、社区活跃度 |

综合推荐指数 = 加权平均（★☆ 形式展示）

## 注意事项

- GitHub Trending 页面有反爬限制，脚本内置 fallback 到 Search API
- API 未认证有 60次/小时 频率限制，`--detail` 每个仓库消耗 1 次
- README 内容仅用于辅助分析，不要整段引用到输出中
- 评分要基于实际数据和分析，不要随意打高分
- 如果抓取失败，可用 web_search 搜索 "github trending today" 补充
