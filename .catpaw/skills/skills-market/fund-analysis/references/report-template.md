# 报告模板说明

本文档说明基金分析报告的标准模板和格式规范。

## 报告结构

完整的基金分析报告包含以下章节:

```
1. 标题
2. 报告生成时间
3. 投资建议摘要
4. 基金基本信息
5. 实时行情
6. 技术面分析
7. 持仓分析
8. 基金经理分析
9. 业绩分析
10. 舆情分析
11. 详细投资建议
12. 免责声明
```

## Markdown格式规范

### 标题层级

```markdown
# 一级标题 (报告标题)
## 二级标题 (章节标题)
### 三级标题 (子章节)
```

### 文本格式

```markdown
**加粗文本**: 用于重要数据和建议
*斜体文本*: 用于强调
`代码文本`: 用于基金代码、数值等
```

### 列表

```markdown
- 无序列表项
  - 嵌套列表项

1. 有序列表项
   1. 嵌套有序列表项
```

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

### 引用

```markdown
> 引用文本
```

### 分隔线

```markdown
---
```

## 各章节格式

### 1. 标题

```markdown
# {基金名称} 基金分析报告
```

### 2. 报告生成时间

```markdown
> 报告生成时间: {YYYY-MM-DD HH:MM:SS}
```

### 3. 投资建议摘要

```markdown
## 💡 投资建议摘要

**核心结论**: {一句话总结}

- **操作建议**: {买入/持有/卖出/观望}
- **理想买点**: {数值}
- **止盈位**: {数值}
- **止损位**: {数值}
```

### 4. 基金基本信息

```markdown
## 📊 基金基本信息

- **基金代码**: {6位数字}
- **基金名称**: {完整名称}
- **基金类型**: {股票型/债券型/混合型等}
- **基金规模**: {X.XX}亿元
- **成立日期**: {YYYY-MM-DD}
- **基金经理**: {姓名}
- **基金公司**: {公司名称}
```

### 5. 实时行情

```markdown
## 📈 实时行情

- **当前净值**: {X.XXXX}
- **日涨跌幅**: {+X.XX%}
- **近7日年化**: {+X.XX%}
```

```markdown
### 技术信号

- {信号2}

### 近期收益率
- **30天**: {+X.XX%}
- **60天**: {+X.XX%}
```

### 7. 持仓分析

```markdown
### 前十大重仓股

### 行业分布
```

```markdown
### 业绩指标
```

### 10. 舆情分析

```markdown
## 📰 舆情分析

- **舆情得分**: {X.X}/100
- **情绪等级**: {强烈正面/正面/中性/负面/强烈负面}
```

```markdown

本报告仅供参考,不构成投资建议。投资有风险,入市需谨慎。

- 本报告基于公开数据分析,可能存在滞后性
- 请根据自身风险承受能力进行投资决策
- 建议咨询专业的投资顾问

**数据来源**: 蛋卷基金API
```

## 数值格式规范

### 净值

- 格式: `X.XXXX` (保留4位小数)
- 示例: `1.2345`

### 百分比

- 正数: `+X.XX%` (带+号)
- 负数: `-X.XX%`
- 保留2位小数

### 金额

- 单位: 亿元
- 格式: `X.XX` (保留2位小数)

### 日期

- 格式: `YYYY-MM-DD`
- 示例: `2024-01-01`

### 时间

- 格式: `YYYY-MM-DD HH:MM:SS`
- 示例: `2024-01-01 12:00:00`

- 正面/优秀: ✅
- 中性/良好: ℹ️

- 错误/失败: ❌

## 颜色和强调

### 重要数据

- 使用加粗 `**` 标记重要数值
- 使用代码 `` `` 标记基金代码和特定术语

### 关键建议

- 核心结论使用加粗
- 操作建议使用加粗
- 买卖点位使用加粗

## 报告示例

完整的报告示例请查看 `examples/` 目录下的 `sample-report.md` 文件。

## 自定义模板

如果需要自定义报告模板,可以修改 `scripts/report_generator.py` 中的对应方法。

### 常见自定义

1. **添加新章节**: 在 `generate()` 方法中添加新的格式化方法
2. **修改章节顺序**: 调整 `generate()` 方法中的调用顺序
3. **修改样式**: 修改对应格式化方法的Markdown生成逻辑
4. **添加图表**: 使用图表生成库生成可视化图表

## 输出文件

- 文件名格式: `fund_analysis_{基金代码}_{时间戳}.md`
- 保存位置: `examples/` 目录
- 编码: UTF-8
