# 基金数据源说明

## 数据源概述

本技能使用蛋卷基金API作为主要数据源,通过pysnowball库进行数据获取。

## 支持的数据类型

### 1. 基金基础信息

**API**: `fund_info(fund_code)`

**返回数据**:
- 基金代码
- 基金名称
- 基金类型
- 基金规模
- 成立日期
- 基金经理姓名
- 基金公司名称

**更新频率**: 每日

### 2. 实时行情

**API**: `fund_detail(fund_code)`

**返回数据**:
- 当前净值
- 日涨跌幅
- 近7日年化收益率

**更新频率**: 交易日收盘后

### 3. 净值历史

**API**: `fund_nav_history(fund_code, page, size)`

**返回数据**:
- 日期
- 净值

**更新频率**: 交易日收盘后

**注意事项**:
- 需要分页获取
- 建议获取最近1年数据

### 4. 持仓数据

**API**: `fund_asset(fund_code)`

**返回数据**:
- 前十大重仓股
- 股票代码
- 持仓比例
- 持仓市值

**更新频率**: 每季度

**注意事项**:
- 数据有1-3个月延迟
- 部分基金可能不提供详细持仓

### 5. 基金经理信息

**API**: `fund_manager(fund_code)`

**返回数据**:
- 基金经理姓名
- 从业年限
- 管理该基金年限
- 管理基金数量

**更新频率**: 每月

### 6. 业绩数据

**API**: `fund_achievement(fund_code)`

**返回数据**:
- 各时间段收益率
- 年化收益率
- 最大回撤
- 同类排名
- 超额收益

**更新频率**: 每日

## 数据质量说明

### 数据可靠性

- ✅ 基础信息: 高度可靠
- ✅ 净值历史: 高度可靠
- ✅ 业绩数据: 高度可靠
- ⚠️ 持仓数据: 有延迟,但准确
- ⚠️ 基金经理信息: 基本可靠

### 数据时效性

| 数据类型 | 更新频率 | 延迟时间 |
|---------|---------|---------|
| 基础信息 | 每日 | 1天 |
| 实时行情 | 交易日收盘后 | 1天 |
| 净值历史 | 交易日收盘后 | 1天 |
| 持仓数据 | 每季度 | 1-3个月 |
| 基金经理 | 每月 | 1个月 |
| 业绩数据 | 每日 | 1天 |

## 缓存策略

为了提高性能和减少API调用,我们实现了多级缓存:

### 内存缓存

- **实时数据**: 缓存5分钟
- **历史数据**: 缓存24小时
- **使用LRU算法**: 自动清理最久未使用的数据

### 缓存键格式

- 基础信息: `basic_info_{fund_code}`
- 实时行情: `realtime_{fund_code}`
- 净值历史: `nav_history_{fund_code}_{days}`
- 持仓数据: `holdings_{fund_code}`
- 基金经理: `manager_{fund_code}`
- 业绩数据: `performance_{fund_code}`

## 错误处理

### 常见错误

1. **基金代码不存在**
   - 错误信息: "基金代码不存在或格式错误"
   - 处理方式: 提示用户检查基金代码

2. **网络连接失败**
   - 错误信息: "网络连接失败"
   - 处理方式: 自动重试3次

3. **API限流**
   - 错误信息: "API请求过于频繁"
   - 处理方式: 增加请求间隔(1秒)

4. **数据格式错误**
   - 错误信息: "数据格式错误"
   - 处理方式: 记录日志,使用默认值
